# Project overview
Assume this is a green field project and no one has used it. No need to preserve old iterations or requirements of this app. All old saves of any data can always be discarded and deprecated with no cutover plan to new versions.

## Coding Standards
- Don't create massive comments. They should be as minimal as possible

## Contributing
- Commits must never have a co-author or commit description. Commit subject should be short and sweet less than 75 characters. Always commit in smaller groups never monolithic commits
- If following the above rule precisely then pushing to main is allowed
